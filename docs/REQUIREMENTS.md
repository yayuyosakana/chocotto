# Chocotto — 要件定義 v1.0

> 今日ふらっと、目的だけで集まって、終わったら解散。名前も知らなくていい。
> 韓国で話題の **“숏셜링(ショートソーシャリング)”** の東京版。

## 0. 背景・狙い（なぜ作るか）

- リサーチで見えた最大のニーズ：**Z世代の孤独**と、対面で会う機会の激減。さらに
  「相手は自分と会いたくないだろう」と誤解して誘えない **empathy perception gap**。
- 解として世界的に来ている **soft socialising（会話必須じゃない、軽い同席）**。
  韓国ではその先の **“숏셜링”**＝目的だけで集まりサッと解散・名前も聞かない使い捨ての集まりが話題。
- 既存（소모임/문토/Trevari/Frip、Timeleft/222/Pie）は **クラブ所属・プロフィール・週次予約・会話前提**。
  → **「今日・単発・無名・フォロー義務ゼロ」は空白**。さらに **日本市場はほぼ空白**＝今が好機。
- 差別化の核：**地域（日本/東京）＋プロダクトの性格（無名×消える×後腐れゼロ）**。
  Hook（集客）は **「韓国で話題の숏셜링、ついに東京で」** を前面に。

## 2. ペルソナ

東京の20〜30代。一人で外出はするが「誰かと居たい、でも会話や継続関係はしんどい」層。
ぼっち作業・朝ラン・展示を“ゆるく誰かと”したい。

## 3. コアメカニクス（場所シェア型）

作成（会場＋時間＋カテゴリ＋定員）→ 近所×今日を発見 → ワンタップ参加 →
消える集合チャット → 到着チェックイン → **自動解散・消滅** → 両者✓の時だけ“また”で連絡（アプリ内のみ）

## 4. 確定した仕様（意思決定ログ）

| 論点 | 決定 |
|---|---|
| 差別化の主役 | 無名 × 消える × 後腐れゼロ |
| 最初のカテゴリ | カフェ/コワーク作業(body doubling)。他カテゴリも選択可 |
| ターゲット | 東京（場所をシェアして集まる方式） |
| 会場の指定 | 地図ピン＋登録スポット候補＋Google Mapsリンク（公共はソフト担保） |
| 鮮度ウィンドウ | 当日〜48h |
| 参加方式 | 即参加ワンタップ（先着） |
| ノーショー対策 | チェックイン＋信頼スコア ＋ 直前キャンセルで一時参加制限 |
| 匿名性の深さ | ニックネーム＋絵文字のみ（顔・本名なし） |
| “また会いたい”再接続 | 両者✓でアプリ内チャット開放（連絡先は渡さない） |
| 集合チャット | 消えるグループチャット（終了で削除） |
| 女性安全 | 性別情報を一切持たない（→ 女性限定モードは無し・要再検討） |
| マネタイズ | 今は決めない（成長優先） |
| 点火（GTM） | あとで考える |
| 定員 | ホストが自由設定（最低3人） |
| ブランド/名前 | 純日本語でゆるく可愛く → **Chocotto** |

## 5. 機能要件（MVP）

- 認証：**匿名サインイン**（メール不要、ニックネーム＋絵文字のオンボーディング）
  - ※本来の要件は本人性のあるOTP。MVPは気楽さ優先で匿名。OTP化は [NEXT_STEPS](./NEXT_STEPS.md) 参照。
- 集まり作成：カテゴリ／会場名／Google Mapsリンク（任意）／日時(今〜48h)／長さ／定員(3〜)
- 発見：今日〜48hの一覧、カテゴリフィルタ、リアルタイム反映
- 参加：ワンタップ／キャンセル（直前は信頼スコア減＋一時制限）
- 詳細：参加者一覧、到着チェックイン（信頼スコア＋）、Google Mapsへの導線
- 集合チャット：参加者のみ・リアルタイム・**終了後に自動消滅**（pg_cron）
- 「また会いたい」：両者✓でマッチ → アプリ内DM（連絡先非交換）
- 通報：各参加者を通報可能
- PWA：インストール可能（manifest / service worker）

## 6. 非機能要件

- **安全**：公共の場所限定（ソフト担保）／3人以上／通報・ブロック／信頼スコア／直前キャンセル制限
- **プライバシー**：会場アンカー型（人の位置は追わない）／チャットは ephemeral
- **コスト**：サーバレス無料枠（Supabase Free + Vercel Hobby）で運用0円スタート
- **セキュリティ**：全テーブル RLS。クライアントは anon/publishable キーのみ使用

## 7. スコープ外（MVP）

決済 / AI性格マッチ / 多都市 / ネイティブアプリ / 友達グラフ・フォロー / Web Push（雛形のみ）

## 8. KPI

- 最重要＝**流動性**：同一エリアで「定員到達→実開催」した数
- 次点：参加→チェックイン率、週次再参加率、“また”相互成立率
